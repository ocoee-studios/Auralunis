# AuraLunis Native / Device Follow-Up Runbook

## Goal
Move the repaired React Native / Expo scaffold onto a real iPhone so the camera, location, compass, motion sensors, haptics, photo saving, and outdoor Sky Lens calibration can be checked honestly.

## Stage 1 — Mac preflight

Open Terminal in the AuraLunis project folder and run:

```bash
chmod +x ./scripts/bootstrap-native-device.sh
./scripts/bootstrap-native-device.sh
```

The script:
1. installs JavaScript dependencies,
2. aligns dependencies to the Expo SDK,
3. installs `expo-dev-client`,
4. runs `expo-doctor`,
5. runs the AuraLunis QA suites.

## Stage 2 — Expo account and internal development build

```bash
npx eas-cli@latest login
npx eas-cli@latest build --platform ios --profile development
```

Install the internal development build on the physical iPhone using the installation link generated by EAS.

## Stage 3 — Start the local development server

```bash
npm run start:dev-client
```

Open the installed AuraLunis development app on the iPhone and connect it to the local development server.

## Stage 4 — In-app device diagnostics

Open:

**Settings → Native Device QA → Open Device Diagnostics**

Run the checks separately:
1. Refresh Permission Status
2. Request Camera
3. Request Location
4. Request Photo Save
5. Test Compass
6. Test Motion Sensors
7. Test Haptic

Expected result:
- camera permission: pass,
- location permission: pass,
- compass heading: pass or calibration warning,
- photo-save permission: pass,
- accelerometer: pass,
- gyroscope: pass,
- magnetometer: pass,
- haptics: confirm by touch.

## Stage 5 — Sky Lens permission fallback

Test both branches:

### Camera allowed
- Open Sky tab.
- Tap Open Sky Lens.
- Approve the camera prompt.
- Confirm Sky Lens placeholder opens.
- Close it and reopen it.

### Camera denied
- Reset camera permission in iPhone Settings if needed.
- Open Sky Lens again.
- Decline the prompt.
- Confirm AuraLunis opens Manual Sky Map instead of a dead end.

## Stage 6 — Outdoor Sky Lens calibration

Go outside with a clear view of the sky.

1. Open Device Diagnostics.
2. Test Compass.
3. If heading accuracy is weak, move the iPhone slowly in a figure-eight.
4. Open Sky Lens.
5. Compare known objects one at a time:
   - Moon
   - Venus if visible
   - Jupiter if visible
   - Polaris
   - Sirius
   - Orion
6. Record the result in `qa/SKY_LENS_OUTDOOR_TEST_LOG_TEMPLATE.md`.

## Stage 7 — Local persistence tests

### Settings
- Change theme to Soft Moon.
- Turn Watch sync off.
- Change Watch face and complication choices.
- Force-close the app.
- Reopen.
- Confirm choices persist.

### Notes and Vault
- Save a Cosmic Note.
- Save a LifeSky moment.
- Save a Deep Sky archive object.
- Force-close the app.
- Reopen.
- Confirm the Vault count persists.
- Clear Prototype Vault in Settings and confirm the count resets.

## Stage 8 — Trial popup

To retest first-open behavior during development, clear app storage or reinstall the development build.

Confirm:
- popup appears on fresh install,
- Continue Exploring dismisses it,
- popup does not reappear after dismissal,
- Start Trial reaches the StoreKit / RevenueCat placeholder,
- Restore Purchases reaches the placeholder.



These remain separate implementation tasks:
- StoreKit / RevenueCat sandbox purchases,
- production astronomy ephemeris and star catalog,
- real AR overlay calculations,
- watchOS target,
- WidgetKit complications,
- notifications,
- production encrypted Vault,
- audio playback,
- optional Oracle backend.


## Stage 9 — RevenueCat sandbox

After App Store Connect and RevenueCat dashboard setup:

1. Add the public RevenueCat iOS SDK key to `app.json`.
2. Rebuild the EAS internal-development app.
3. Reinstall the development build.
4. Use an App Store sandbox account.
5. Test Horizon+ monthly and annual.
6. Test Aura Pro monthly and annual.
7. Cancel one purchase flow.
8. Restore purchases.
9. Open Settings → Manage Subscription.
10. Confirm Sovereign stays waitlist-only.

Use `qa/PAYWALL_REVENUECAT_QA_CHECKLIST.md`.
